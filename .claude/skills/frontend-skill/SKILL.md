## Instructions

1. **Page Structure**
   - Create semantic HTML structure
   - Organize sections logically
   - Ensure accessibility basics

2. **Components**
   - Build reusable UI components
   - Separate components by responsibility
   - Use props / parameters where applicable

3. **Layout**
   - Use Flexbox and Grid
   - Responsive layouts for all screen sizes
   - Consistent spacing and alignment

4. **Styling**
   - Apply clean and consistent styling
   - Use CSS / Tailwind / Styled Components
   - Follow design system or theme
   - Handle hover, focus, and active states

## Best Practices
- Mobile-first approach
- Reusable and modular components
- Consistent colors and typography
- Avoid inline styles where possible
- Optimize for performance and readability
